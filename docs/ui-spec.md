## 4. ログイン（`/login`）

- 入力
- `メールアドレス`
- `パスワード`
- 操作
- `ログイン`
- `パスワードを再設定する`（`/password-reset`）
- サインアップ導線は表示しない（MVP対象外）
- 未ログイン状態で `/dashboard` などへ直接遷移した場合は `/login` へリダイレクトする

## 4.1 パスワード再設定（`/password-reset`）

- 目的
- Cognito のパスワード再設定フローに対応する
- 画面モード
- 新しいパスワード必須（`CONFIRM_SIGN_IN_WITH_NEW_PASSWORD_REQUIRED`）
  - 入力: 新しいパスワード / 確認
  - 操作: `設定してログイン`
- メール再設定
  - 入力: メールアドレス
  - 操作: `確認コードを送信`
  - 送信後入力: 確認コード / 新しいパスワード / 確認
  - 操作: `パスワードを再設定`
- 導線
- `ログイン画面へ戻る`

## 5. Dashboard（`/dashboard`）

## 6. トレーニング実施画面（`/training-session`）

### 6.1 画面構成

- ヘッダー
- タイトル: `トレーニング実施`
- 日付表示
- `下書きをクリア` ボタン（全種目の下書きを破棄）
- 下書き状態表示（下書きがある場合のみ `下書き保存中: ...`）
- 画面下部固定: `記録して終了`

### 6.2 種目カード

- 表示要素
- `優先 {n}`
- 種目名表示: `trainingName : bodyPart`（`bodyPart` が空の場合は `trainingName` のみ）
- `直近`（最新実績、なければ `未実施（メニュー既定値...）`）
- 操作
- `前回と同じ`: 最新実績を入力欄へ反映
- `入力クリア`: 当該種目を今回下書きから削除
- 入力欄
- `重量`（`step=0.01`、空欄許可、placeholderに既定値）
- `回数 ({min}回 - {max}回)`（整数、空欄許可、`min/max` は `TrainingMenuItem.defaultRepsMin/defaultRepsMax` を表示）
- `セット`（整数、空欄許可、placeholderに既定値）
- 任意入力
- `セット詳細を入力` / `セット詳細を閉じる`
- セット詳細は各setごとに重量/回数を入力

### 6.3 入力状態ルール

- `空欄`: 未入力
- `0`: 無効値（保存対象外）
- `入力クリア`: 当該種目をドラフトから削除
- 正式保存対象は `重量/回数/セット` がすべて正の値の種目のみ
- `記録して終了` で保存対象が0件の場合は保存せずメッセージ表示

## 7. トレーニングメニュー管理（`/training-menu`）

### 7.1 メニュー一覧

- ヘッダー: `トレーニングメニュー`
- `AIでメニュー生成` ボタン（`/training-menu/ai-generate`）

### 7.2 新規追加

- 入力項目
- `トレーニング名`
- `鍛える部位`（任意、例: 胸 / 背中 / 脚）
- `重量(kg)`（小数）
- `回数 最小`（整数）
- `回数 最大`（整数）
- `セット`（整数）
- `追加` ボタン

### 7.3 既存項目編集

- 各カードに表示
- `順序 {order}`
- 並び替え: `↑` / `↓`
- `削除`
- 編集入力: `トレーニング名` / `鍛える部位` / `重量` / `回数 最小` / `回数 最大` / `セット`

注記:
- 並び替えはドラッグ&ドロップではなく `↑` / `↓` ボタン方式。

## 10. Daily（`/daily/:date`）

- ヘッダー
- タイトル `Daily`
- `カレンダー` / `トレーニング開始` 導線
- `保存` ボタン
- 保存状態表示（`保存中` / `最終保存` / `未保存`）
- 入力は自動保存（3秒デバウンス）し、`保存` ボタンで即時保存可能
- 体重・体脂肪率
- `体重(kg)` / `体脂肪率(%)` / `測定時刻` を1行3列表示
- 体調
- 5段階アイコンの1タップ入力
- コメント
- 日記
- 自由記述
- その他トレーニング
- 文字列追加 / 削除
- 当日の筋トレ内容
- 当日 `GymVisit` の種目を統合した番号付き一覧（1, 2, 3...）
  - 種目名表示形式: `トレーニング名 : 部位`（部位未設定時はトレーニング名のみ）

## 13. 日時表示・入力フォーマット

- 画面表示
- 日付: `YYYY/MM/DD`
- 時刻: `HH:mm`
- 測定時刻入力: `input[type=time]`
- API/保存側
- 日付: `YYYY-MM-DD`
- 時刻: RFC3339
- トレーニング実施時刻はバックエンドで RFC3339 UTC（秒精度）保存
- 画面表示・AI利用時は `timeZoneId` 基準ローカル時刻へ変換

## 14. PWA（iPhoneホーム画面追加）

- `manifest.webmanifest` を配信し、ホーム画面追加に対応する。
- iOS向けメタタグを設定する。
- `apple-mobile-web-app-capable: yes`
- `apple-mobile-web-app-title: KinTrain`
- `apple-touch-icon: /icons/icon-180.png`
- 起動モードは `standalone` とし、ホーム画面起動時に単独アプリ風表示とする。
- 現時点ではService Worker未導入のため、オフライン利用は対象外とする。

## 15. 受け入れ基準（実装準拠）

- 未ログイン時は `/login` のみアクセス可能であること。
- `/login` でメールアドレス/パスワード入力後にアプリへ遷移できること。
- `/password-reset` でパスワード再設定操作ができること。
- `/dashboard` から `トレーニング開始` と `本日の日記をつける` に遷移できること。
- `/training-session` で `前回と同じ` / `入力クリア` / `セット詳細` が機能すること。
- `/training-session` / `/daily/:date` / `/dashboard` の種目名が `トレーニング名 : 部位` 形式で表示されること（部位未設定時はトレーニング名のみ）。
- `/training-session` の入力はリロード後も復元されること（localStorage）。
- `記録して終了` で保存され、`/daily/:date` で内容確認できること。
- `/training-menu` で追加/編集/削除/上下並び替えができること。
- `/training-menu/ai-generate` で提案反映できること。
- `/calendar` で実施日と体調アイコンを確認し、日付遷移できること。
- `/daily/:date` で体重・体脂肪率・測定時刻・体調・日記・その他トレーニングを更新できること。
- `/daily/:date` は自動保存（3秒デバウンス）と保存ボタンの両方で更新できること。
- `/settings` でユーザ情報とAIキャラクター名/口調を更新できること。
- `/settings` からログアウトでき、`/login` へ遷移すること。
- `/ai-chat` でモックストリーミング応答が表示されること。
- iPhoneでホーム画面に追加した場合、standalone起動できること。
