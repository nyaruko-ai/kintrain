# KinTrain UI詳細設計（MVP）

最終更新日: 2026-02-28
対象: スマホ優先（PCレスポンシブ対応）

## 1. 設計方針

- 最優先は操作性（ジム現場で最短入力）。
- 1画面で「参照（直近実績）+入力（重量/回数/セット）」を完結させる。
- 途中保存（ドラフト）と正式保存（記録確定）を分離する。
- 日次画面（Daily）を中心に、カレンダーとトレーニング記録を接続する。
- 日時はTimeZoneを明示し、AI/ユーザーで解釈差が出ない表示に統一する。

## 2. 画面一覧

- `/dashboard`
- `/training-session`（ジム実施中の入力画面）
- `/training-menu`（メニュー管理）
- `/training-menu/ai-generate`（AI生成ウィザード）
- `/calendar`（月次カレンダー）
- `/daily/:date`（日次詳細/入力）
- `/history`
- `/progress`
- `/ai-chat`

## 3. ジム利用時フロー

1. ジム到着後、`/dashboard` を開く。
2. `トレーニング開始` ボタンを押して `/training-session` へ遷移。
3. 優先順で並んだトレーニング一覧から、実施できるものを上からこなす。
4. 実施した行に `重量/回数/セット` を入力する。
5. 必要時のみ `セット詳細` を開いてセット別入力する（初期非表示）。
6. 満足した時点で `記録して終了` を押す。
7. 正式記録後、当日の `Daily` へ反映する。

## 4. Dashboard（/dashboard）

### 4.1 主要アクション

- `トレーニング開始`（/training-session）
- `本日の日記をつける`（/daily/{today}）

### 4.2 表示情報

- 直近トレーニング実施日
- 今日の体調（未入力なら入力促進）
- 当日の簡易サマリ（入力済み日記/体重/体脂肪率）
- 今日の基準時刻（例: `2026/02/28 19:05 JST`）

## 5. トレーニング実施画面（/training-session）

### 5.1 一覧の表示

- 表示対象は「現在のトレーニングメニュー」全体（1日限定メニューではない）。
- 各行に以下を表示:
- 種目名（マシン名）
- 優先順位番号
- 直近実績（重量/回数/セット）
- 入力欄（重量/回数/セット）
- `前回と同じ` ボタン（1クリック入力）
- `セット詳細` トグル（初期非表示）

### 5.2 優先順位ロジック

- 履歴ベースで並び替える。
- 昨日実施種目は上位に来ないよう強く減点する。
- 未実施期間が長い種目を加点する。
- 当日実施済みは下位へ移す。
- 同点時はメニュー手動順序を優先する。

### 5.3 入力と保存

- 入力変更はドラフトへ即時保存（デバウンス300ms）。
- リロード後もドラフト復元する。
- `記録して終了` 時のみ `GymVisit` として正式保存する。

## 6. カレンダー画面（/calendar）

### 6.1 月表示

- 1ヶ月単位で表示する。
- カレンダー上では「筋トレ実施日」のみ識別しやすく表示する。
- 各日付セルに体調評価（1〜5）をアイコン表示する。

### 6.2 日付選択

- 日付タップで `/daily/:date` へ遷移する。

## 7. Daily画面（/daily/:date）

### 7.1 表示項目

- 当日の体重・体脂肪率・記録時刻
- 当日の筋トレ内容（GymVisit要約）
- 当日の日記
- 当日の体調（5段階）
- その他トレーニング（フリー入力、例: ジョギング1km）

### 7.2 入力/更新

- すべて後から更新可能。
- 体調は以下で入力:
- 5段階アイコンの1タップ即記録
- 任意コメント入力
- 日記は自由記述。
- その他トレーニングは自由記述を複数追加可能。

### 7.3 筋トレ内容との関係

- 筋トレ内容は当日 `GymVisit` から自動表示する。
- 必要時は該当 `GymVisit` 編集導線で更新できる。

## 8. トレーニングメニュー管理（/training-menu）

### 8.1 手動管理

- 追加/更新/削除
- 並び替え（ドラッグ&ドロップ）
- 入力項目:
- `machineName`
- `defaultWeightKg`
- `defaultReps`
- `defaultSets`

### 8.2 AI生成

- `AIでメニュー生成` を提供。
- 生成前ヒアリング:
- 利用可能マシン情報（ジム設備ページURLまたは貼り付け）
- マシンのみ / フリーウェイト併用
- 目標（筋肥大/減量/維持）
- 週間頻度・時間制約
- 生成結果は差分プレビューで確認後に採用する。

## 9. AIチャット（/ai-chat）

- AIエージェントの役割名は固定で `AIコーチ` とする。
- トレーニング全般相談を可能にする。
- AI提案からメニュー更新を実行可能にする。
- ただし即時反映せず、確認後に反映する。
- 応答はストリーミング表示する。

### 9.1 AIキャラクター設定

- 設定可能項目:
- `characterId`（MVP既定: `nyaruko`）
- キャラクター名
- キャラクターアイコン（プリセット選択または画像URL指定）
- 口調プリセット（丁寧/フレンドリー/コーチ風）
- `expressions`（`default|thinking|surprised|love|doubt|angry`）
- 設定導線:
- `/ai-chat` 右上の `キャラクター設定` から編集
- 初期ロード順:
- `GET /ai-character-profile` の値
- 未設定時は `assets/characters/nyaruko/character-profile.json`
- 保存時は `PUT /ai-character-profile` でユーザー単位に永続化する。

### 9.2 キャラクター演出

- チャットヘッダーは `AIコーチ（{characterName}）` 形式で表示する（例: `AIコーチ（ニャル子）`）。
- AIメッセージバブルにキャラクターアイコンと名前を表示する。
- ストリーミング中はキャラクターのタイピングインジケータを表示する。
- ユーザーメッセージとAIメッセージを視覚的に明確分離する。
- 表情切り替え:
- 既定は `expressions.default`
- ストリーミング中は `expressions.thinking`
- Runtimeイベントに `expressionKey` が含まれる場合は対応表情を優先する（未指定時は `default`）。

## 10. 日時表示・入力フォーマット

- TimeZone ID: `Asia/Tokyo`（将来ユーザー設定で変更可能）
- 画面表示:
- 日付: `YYYY/MM/DD`
- 時刻: `HH:mm`
- タイムゾーン略称付き: 例 `2026/02/28 19:05 JST`
- API送受信:
- 日付のみ: `YYYY-MM-DD`
- 時刻付き: RFC3339（UTCまたはオフセット付き）
- `Daily` の日付判定は `timeZoneId` 基準で行う。

## 11. 操作性要件（UX）

- 主要アクションは下部固定で片手操作しやすくする。
- 数値入力はテンキーを使用する。
- 主要操作は3タップ以内を目標にする。
- 保存完了はトースト表示で即時フィードバックする。
- 未保存入力がある場合は離脱確認を表示する。

## 12. モダンデザイン要件

- 明確なタイポグラフィ階層を持つ。
- 状態色（未実施/実施中/完了、体調1〜5）を一貫化する。
- カードレイアウトで一覧可読性を高める。
- ジム環境で見やすい高コントラストを維持する。

## 13. 受け入れ基準（UI）

- `トレーニング開始` から実施入力まで迷わず到達できること。
- 一覧上で `重量/回数/セット` を即入力できること。
- `前回と同じ` 1クリック入力が動作すること。
- セット詳細は初期非表示で必要時のみ開くこと。
- リロード後に途中入力が復元されること。
- `記録して終了` でのみ正式保存されること。
- カレンダーで筋トレ実施日と体調アイコンを確認できること。
- 日付タップで `Daily` へ遷移できること。
- `Daily` で体重・体脂肪率・日記・体調・その他トレーニングを更新できること。
- 体調アイコン1タップ記録が動作すること。
- AIキャラクター設定（名前/アイコン/口調）が変更できること。
- AIチャットでキャラクターが話している演出（アイコン表示）が機能すること。
- 全画面で日時がTimeZone付きで一貫表示されること。
