- `メールアドレス`
- `パスワード`

- `ログイン`
- サインアップ導線は表示しない（MVP対象外）

## 5. Dashboard（`/dashboard`）

- Heroカード
- 表示: `Today`
- アクション: `トレーニング開始`（`/training-session`）
- アクション: `本日の日記をつける`（`/daily/{today}`）
- `今日の状態` カード
- 表示: 体重 / 体脂肪率 / 体調（未入力時は `未入力`）
- `直近の筋トレ` カード
- 表示: 最新 `GymVisit` の日付と最大4件の種目要約
- 導線: `カレンダーで確認`

## 6. トレーニング実施画面（`/training-session`）

### 5.1 画面構成

- ヘッダー
- タイトル: `トレーニング実施`
- 日付表示
- `下書きをクリア` ボタン（全種目の下書きを破棄）
- 下書き状態表示
- 下書きがある場合: `下書き保存中: ...`
- ない場合: `まだ入力はありません。...`
- 画面下部固定: `記録して終了`

### 5.2 種目カード

- 表示要素
- `優先 {n}`
- `trainingName`
- `直近`（最新実績、なければ `実績なし`）
- 操作
- `前回と同じ`: 最新実績を入力欄へ反映
- `入力クリア`: 当該種目を今回下書きから削除
- 入力欄
- `重量`（`step=0.01`、空欄許可、placeholderに既定値）
- `回数`（整数、空欄許可、placeholderに既定値）
- `セット`（整数、空欄許可、placeholderに既定値）
- 任意入力
- `セット詳細を入力` / `セット詳細を閉じる`
- セット詳細は各setごとに重量/回数を入力

### 5.3 入力状態ルール

- `空欄`: 未入力
- `0`: 無効値（保存対象外）
- `入力クリア`: 当該種目をドラフトから削除
- 正式保存対象は `重量/回数/セット` がすべて正の値の種目のみ
- `記録して終了` で保存対象が0件の場合は保存せずメッセージ表示

## 7. トレーニングメニュー管理（`/training-menu`）

### 6.1 メニュー一覧

- ヘッダー: `トレーニングメニュー`
- `AIでメニュー生成` ボタン（`/training-menu/ai-generate`）

### 6.2 新規追加

- 入力項目
- `トレーニング名`
- `重量(kg)`（小数）
- `回数`（整数）
- `セット`（整数）
- `追加` ボタン

### 6.3 既存項目編集

- 各カードに表示
- `順序 {order}`
- 並び替え: `↑` / `↓`
- `削除`
- 編集入力: `トレーニング名` / `重量` / `回数` / `セット`

注記:
- 並び替えはドラッグ&ドロップではなく `↑` / `↓` ボタン方式。

## 8. AIメニュー生成（`/training-menu/ai-generate`）

- タイトル: `AIメニュー生成（モック）`
- 入力項目
- 方針: `マシンのみ` / `マシン + フリーウェイト`
- 目標: `筋肥大` / `減量` / `維持`
- 週間頻度
- 表示
- 提案プレビュー一覧
- 現行メニュー一覧
- 操作
- `この提案でメニュー更新`
- `戻る`

## 9. カレンダー（`/calendar`）

- 月移動: `前月` / `次月`（`month=YYYY-MM`）
- 日付セル表示
- 実施日: ドット表示
- 体調: 1〜5のアイコン（未入力は `○`）
- 日付タップで `Daily` へ遷移

## 10. Daily（`/daily/:date`）

- ヘッダー
- タイトル `Daily`
- `カレンダー` / `トレーニング開始` 導線
- 体重・体脂肪率
- `体重(kg)` / `体脂肪率(%)` / `測定時刻`
- 体調
- 5段階アイコンの1タップ入力
- コメント
- 日記
- 自由記述
- その他トレーニング
- 文字列追加 / 削除
- 当日の筋トレ内容
- 当日 `GymVisit` 一覧
- 各visitに時刻帯（開始-終了）と種目明細

## 11. AIチャット（`/ai-chat`）

- ヘッダー表示
- ラベル: `AI コーチ`
- 表示名: `{characterName}`（既定: ニャル子）
- アイコン: `avatarImageUrl`（default固定）
- チャット本文
- AI/ユーザの吹き分け表示
- AIメッセージにアイコンと名前表示
- 入力
- textarea + `送信`
- モックストリーミング
- 応答テキストを分割し逐次表示

注記:
- 現モックのAIチャットからトレーニングメニュー更新は実行しない。

## 12. ユーザ設定（`/settings`）

- ユーザ設定項目
- `ユーザ名`
- `性別`
- `生年月日`
- `身長`
- `タイムゾーン`
- `保存` ボタン（モック表示）
- AIコーチキャラクター設定
- `キャラクター名`
- `口調プリセット`
- アイコンは default 固定（選択UIなし）
- アカウント操作
- `ログアウト`

## 13. 日時表示・入力フォーマット

- 画面表示
- 日付: `YYYY/MM/DD`
- 時刻: `HH:mm`
- 測定時刻入力: `input[type=time]`
- API/保存側
- 日付: `YYYY-MM-DD`
- 時刻: RFC3339
- トレーニング実施時刻はバックエンドで RFC3339 UTC（秒精度）保存
- 画面表示・AI利用時は `timeZoneId` 基準ローカル時刻へ変換

## 14. 受け入れ基準（モック準拠）

- 未ログイン時は `/login` のみアクセス可能であること。
- `/login` でメールアドレス/パスワード入力後にアプリへ遷移できること。
- `/dashboard` から `トレーニング開始` と `本日の日記をつける` に遷移できること。
- `/training-session` で `前回と同じ` / `入力クリア` / `セット詳細` が機能すること。
- `/training-session` の入力はリロード後も復元されること（localStorage）。
- `記録して終了` で保存され、`/daily/:date` で内容確認できること。
- `/training-menu` で追加/編集/削除/上下並び替えができること。
- `/training-menu/ai-generate` で提案反映できること。
- `/calendar` で実施日と体調アイコンを確認し、日付遷移できること。
- `/daily/:date` で体重・体脂肪率・測定時刻・体調・日記・その他トレーニングを更新できること。
- `/settings` でユーザ情報とAIキャラクター名/口調を更新できること。
- `/settings` からログアウトでき、`/login` へ遷移すること。
- `/ai-chat` でモックストリーミング応答が表示されること。
